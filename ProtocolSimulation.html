Simulation of Reliable Transport Protocols


In this lab, we will incrementally develop and simulate protocols for reliable transport as described in Chapter 3 of the textbook by Kurose and Ross 6th ed. We will make use of SimPy, a popular open-source Python package for discrete-event simulations.

Requirements

  • Python3 (version >3.2)

  • SimPy (version >=3.0.10) for Python3.
    In Linux, this can be installed using the command $ pip3 install simpy . You can check if you have the correct version installed, using the following command (for Linux): $ python3 -c “import simpy; print(simpy.__version__)” . It should output the current version of SimPy (for example “3.0.10”)

Template

To get started, here is a template that contains models for the sending and receiving processes, the sender and receiver-side protocols, an unreliable channel, and a Testbench to connect these together and run the simulation. The names for the entities and their interfaces closely follows the terminology used in K&R.

 

  • Download all of the following files into a single folder:

    • Packet.py

    • TestPacket.py

    • Applications.py

    • Channel.py

    • Protocol_rdt1.py

    • Protocol_rdt2.py

    • Testbench.py

  • Check if the simulation works by running the command $ python3 Testbench.py

  • To understand the models, it is suggested to go through the code in the same order as the list above.
